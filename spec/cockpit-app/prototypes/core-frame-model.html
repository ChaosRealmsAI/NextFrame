Core Walkthrough 01

Frame-Pure Rendering Model

Think of the timeline like a music score: at any beat position, there is exactly one set of notes playing. In NextFrame, that means one exact set of clips is active at time t. Feed that set into renderFrame(t), and you get one deterministic visual output.

The Invariant

Timeline -> Active Clips -> Frame
1. Timeline stores scheduled clips Each clip has a start and duration. Nothing is guessed at runtime.
→
2. renderFrame(t) Checks which clips satisfy start <= t < end.
→
3. One exact frame The same time value always produces the same visual stack.
Music Score Analogy
Beat 4.80s

You do not ask, "what might be playing around here?" You ask, "what notes are playing on this beat?" The answer is precise, layered, and repeatable.

Why AI Should Care
No hidden frame state

If the skeleton is pure, AI can reason from timeline data alone. It does not need to chase mutable editor ghosts to predict what a frame should look like.
