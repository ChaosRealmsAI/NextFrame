Video Transition Types

8 categories, 25+ transitions, live CSS/JS demos. Click "Play" to see each one in action.

1. Cut Transitions (硬切)

90% of video editing is hard cuts. Other transitions are for emphasis. The cut is the default.

Hard Cut

Scene A instantly becomes Scene B. No animation, no effect. Zero frames of transition.

When to use: Conversation shot/reverse-shot, fast-paced editing, most of the time.

Scene AInterview shot
Scene BReverse angle
// Hard Cut: instant swap, no transition sceneA.style.opacity = '0'; sceneB.style.opacity = '1'; // That's it. No animation. The cut IS the absence of transition.

Jump Cut

Same scene, slight time skip. The subject shifts position abruptly. Used in vlogs, montages.

When to use: Time compression, energetic pacing, removing pauses in talking-head videos.

👤
Same scene, subject shifts
150ms
// Jump Cut: same scene, different position // Flash white briefly to sell the "skip" stage.style.background = '#fff'; subject.style.transform = `translate(${randomX}px, ${randomY}px)`; setTimeout(() => stage.style.background = '', 50);

Match Cut

Similar shapes or colors bridge two scenes. A ball becomes a planet, a wheel becomes a clock.

When to use: Thematic connection, storytelling, the "artistic" cut.

Moon
800ms
// Match Cut: morph shape A into shape B // Keep the circle, change color + label shape.style.transition = `all ${duration}ms cubic-bezier(0.16,1,0.3,1)`; shape.style.background = sceneB_color; shape.textContent = 'Planet'; // The shape continuity tricks the eye into accepting the scene change.

2. Dissolve Transitions (溶解)

Just opacity animation on two layers. The workhorse of emotional transitions.

Cross Dissolve

A fades out while B fades in, overlapping. Both visible simultaneously during the transition.

When to use: Passage of time, emotional shift, dream sequences.

Scene APresent day
Scene BMemory
1000ms
/* Cross Dissolve: pure CSS opacity */ .scene-a { transition: opacity ${duration}ms ease; opacity: 0; /* fade out */ } .scene-b { transition: opacity ${duration}ms ease; opacity: 1; /* fade in simultaneously */ } /* Key: both layers animate at the same time, creating overlap */

Fade to Black

A fades to black, pause, then B fades in from black. Sequential, not simultaneous.

When to use: End of a chapter, dramatic pause, "time passes" feeling.

Scene AEnd of act
Scene BNew chapter
800ms
// Fade to Black: A → black → B (sequential) overlay.style.transition = `opacity ${half}ms ease`; overlay.style.opacity = '1'; // fade to black setTimeout(() => { sceneA.style.opacity = '0'; sceneB.style.opacity = '1'; // swap underneath overlay.style.opacity = '0'; // reveal B }, half);

Fade to White

Same as fade to black, but through white. Brighter, more hopeful feeling.

When to use: Flashbacks, heavenly/dreamy scenes, positive emotional transitions.

Scene ABefore
Scene BAfter
800ms
// Fade to White: identical to fade-to-black, just change overlay color overlay.style.background = '#fff'; // Same timing logic as fade-to-black.

Dip to Color

Same pattern as fade-to-black/white, but uses a brand color as the intermediate.

When to use: Branded content, social media intros, logo reveals.

Scene AYour brand
Scene BNew segment
800ms
// Dip to Color: overlay.background = brandColor // Same sequential fade pattern, any color works.

3. Wipe Transitions (擦除)

clip-path is the key CSS property. Scene B is revealed by animating a clipping region.

Horizontal Wipe

Scene B is revealed left-to-right (or right-to-left) like a curtain being drawn.

When to use: Scene changes in documentaries, news segments, before/after comparisons.

Scene ABefore
Scene BAfter
800ms
/* Horizontal Wipe: clip-path inset animation */ .scene-b { clip-path: inset(0 100% 0 0); /* fully clipped = hidden */ transition: clip-path ${duration}ms cubic-bezier(0.16,1,0.3,1); } /* trigger: */ sceneB.style.clipPath = 'inset(0 0% 0 0)'; /* fully revealed */ /* Reverse: inset(0 0 0 100%) for right-to-left */

Vertical Wipe

Top-to-bottom reveal. Same technique, different axis.

When to use: Vertical scrolling content, social media transitions.

Scene ATop
Scene BBottom
800ms
/* Vertical Wipe */ .scene-b { clip-path: inset(0 0 100% 0); } /* hidden */ sceneB.style.clipPath = 'inset(0 0 0% 0)'; /* revealed top-to-bottom */

Iris Wipe

Circle expanding from center to reveal Scene B. Classic film technique (Looney Tunes ending).

When to use: Retro feel, comedy, ending/opening focus on a subject.

Scene AFocus point
Scene BRevealed
800ms
/* Iris Wipe: circle() clip-path */ .scene-b { clip-path: circle(0% at 50% 50%); } sceneB.style.transition = `clip-path ${duration}ms cubic-bezier(0.16,1,0.3,1)`; sceneB.style.clipPath = 'circle(75% at 50% 50%)'; /* 75% radius covers the full rectangle including corners */

Clock Wipe

A rotating hand sweeps around from 12 o'clock, revealing Scene B like a clock face.

When to use: Time-related transitions, retro/classic film.

Scene ABefore
Scene BAfter
1200ms
// Clock Wipe: animated polygon via JS requestAnimationFrame // CSS clip-path polygon can't animate between different vertex counts, // so we compute polygon points per frame: function clockFrame(progress) { // Build a polygon from center to 12 o'clock, sweeping clockwise const angle = progress * 360; const points = [{ x: 50, y: 50 }, { x: 50, y: 0 }]; // Add corner points as the sweep passes them if (angle > 45) points.push({ x: 100, y: 0 }); if (angle > 135) points.push({ x: 100, y: 100 }); if (angle > 225) points.push({ x: 0, y: 100 }); if (angle > 315) points.push({ x: 0, y: 0 }); // Add the current sweep point const rad = (angle - 90) * Math.PI / 180; points.push({ x: 50 + 70 * Math.cos(rad), y: 50 + 70 * Math.sin(rad) }); return `polygon(${points.map(p => p.x+'% '+p.y+'%').join(', ')})`; }

Star Wipe ⭐

A star shape expands from center. Famously cheesy. Only use ironically (or if you're SpongeBob).

When to use: Comedy, ironic retro, children's content. Never in serious work.

Scene ASerious
Scene BLOL
1000ms
// Star Wipe: 5-pointed star polygon, scaled from 0 to full function starPoints(scale) { const pts = []; for (let i = 0; i < 10; i++) { const angle = (i * 36 - 90) * Math.PI / 180; const r = i % 2 === 0 ? 50 * scale : 20 * scale; pts.push({ x: 50 + r * Math.cos(angle), y: 50 + r * Math.sin(angle) }); } return `polygon(${pts.map(p=>p.x+'% '+p.y+'%').join(', ')})`; }

4. Slide / Push Transitions (推移)

transform: translateX/Y + positioning. The timing relationship between A and B defines the variant.

Slide

Scene B slides over A from a direction. A stays still underneath.

When to use: App-like navigation, panel reveals, mobile UI transitions.

Scene ACurrent view
Scene BNew view
600ms
/* Slide: B slides over A (A stays put) */ .scene-b { transform: translateX(100%); /* starts off-screen right */ transition: transform ${duration}ms cubic-bezier(0.16,1,0.3,1); } sceneB.style.transform = 'translateX(0%)'; /* slides into view */ /* A doesn't move — B covers it */

Push

A is pushed out while B pushes in. Both move simultaneously in the same direction.

When to use: Carousel-like navigation, equal-weight scene changes.

Scene ALeaving
Scene BArriving
600ms
/* Push: both A and B move together */ .scene-a { transition: transform ${duration}ms ease; } .scene-b { transform: translateX(100%); transition: transform ${duration}ms ease; } /* trigger: */ sceneA.style.transform = 'translateX(-100%)'; /* A exits left */ sceneB.style.transform = 'translateX(0%)'; /* B enters from right */

Cover

B slides over A from a direction. Same as Slide but emphasizes B covering A. A is static.

When to use: Overlay panels, "new content arriving" feel.

Scene ABackground
Scene BCovering
600ms
/* Cover: B slides from bottom over A */ sceneB.style.transform = 'translateY(100%)'; /* starts below */ sceneB.style.transition = `transform ${duration}ms ease`; sceneB.style.transform = 'translateY(0%)'; /* covers A */

Reveal

A slides away to reveal B sitting underneath. B was there all along.

When to use: "Unveiling" content, peeling back layers, dramatic reveals.

Scene BWas here all along
Scene ASliding away
600ms
/* Reveal: A slides away, B is underneath */ /* B is positioned behind A (lower z-index or painted first) */ sceneA.style.transition = `transform ${duration}ms ease`; sceneA.style.transform = 'translateX(-100%)'; /* A exits, revealing B */

5. Zoom Transitions (缩放)

transform: scale() + opacity combo. Creates depth and camera-like movement.

Zoom In

Scene A zooms in (scales up) until it fills/blurs, then Scene B appears at normal scale.

When to use: Focusing on detail, dramatic emphasis, "entering" a scene.

Scene AZooming in...
Scene BArrived
800ms
/* Zoom In: scale A up + fade, then show B */ .scene-a { transition: transform ${duration}ms ease, opacity ${half}ms ease ${half}ms; transform: scale(3); opacity: 0; } .scene-b { transition: opacity ${half}ms ease ${half}ms; opacity: 1; transform: scale(1); /* B appears at normal scale */ }

Zoom Out

Camera pulls back from A, revealing B as the wider context.

When to use: "Big picture" reveal, pulling back to show context, endings.

Scene AClose up
Scene BBig picture
800ms
/* Zoom Out: A shrinks, B appears behind */ sceneA.style.transition = `transform ${d}ms ease, opacity ${d}ms ease`; sceneA.style.transform = 'scale(0.3)'; sceneA.style.opacity = '0'; sceneB.style.transition = `opacity ${d}ms ease`; sceneB.style.opacity = '1';

Zoom Through

Zoom into a detail of A, keep zooming, emerge in B. Continuous motion through both scenes.

When to use: Seamless scene connection, travel/journey montages.

Scene AZooming through...
Scene BEmerged
1000ms
/* Zoom Through: A scales up past view, B scales up from small */ // Phase 1: A zooms in sceneA.style.transform = 'scale(5)'; sceneA.style.opacity = '0'; // Phase 2: B zooms from small to normal (starts before A finishes) setTimeout(() => { sceneB.style.opacity = '1'; sceneB.style.transform = 'scale(1)'; }, duration * 0.3);

6. 3D Transitions (三维)

CSS 3D transforms with perspective. Use sparingly -- flashy but distracting in most contexts.

Cube Rotate

Scenes on faces of a rotating cube. The stage rotates to show the next face.

When to use: Tech demos, gaming content, "cool factor" transitions. Sparingly.

Scene AFront face
Scene BRight face
800ms
/* Cube Rotate: CSS 3D */ .stage { perspective: 800px; } .cube { transform-style: preserve-3d; transition: transform ${duration}ms cubic-bezier(0.16,1,0.3,1); } .face-a { transform: translateZ(100px); } .face-b { transform: rotateY(90deg) translateZ(100px); } /* Rotate the whole cube: */ cube.style.transform = 'translateZ(-100px) rotateY(-90deg)';

Flip

Page flip effect. A rotates on Y-axis, revealing B on the "back side".

When to use: Card reveals, before/after, page-turn metaphors.

Scene AFront
Scene BBack
800ms
/* Flip: backface-visibility is the key */ .wrapper { transform-style: preserve-3d; } .scene-a { backface-visibility: hidden; } .scene-b { backface-visibility: hidden; transform: rotateY(180deg); } /* Flip: */ wrapper.style.transform = 'rotateY(180deg)'; /* The backface-visibility: hidden hides each face when rotated past 90deg */

Fold

Scene A folds away like paper being folded, revealing B behind it.

When to use: Origami/paper metaphors, creative reveals.

Scene BRevealed
Scene AFolding...
800ms
/* Fold: rotateY with transform-origin on one edge */ .scene-a { transform-origin: left center; transition: transform ${duration}ms ease; } sceneA.style.transform = 'rotateY(-90deg)'; /* folds away to the left */

7. Morphing Transitions (变形)

The hardest to implement but most impressive. Shape and color interpolation between scenes.

Shape Morph

An element in Scene A morphs into a different element in Scene B. Requires matching geometry.

When to use: Brand identity transitions, logo animations, premium content.

Square
1000ms
/* Shape Morph: animate border-radius, size, color */ element.style.transition = `all ${duration}ms cubic-bezier(0.16,1,0.3,1)`; element.style.borderRadius = '50%'; // square → circle element.style.background = sceneB_color; element.style.width = '120px'; element.style.height = '120px'; /* For complex shapes: use clip-path or SVG path morphing (SMIL/GSAP) */

Color Morph

The entire color palette shifts from Scene A's palette to Scene B's. Background, accents, all change.

When to use: Mood changes, day-to-night, seasonal transitions.

Day Scene Warm colors
1000ms
/* Color Morph: CSS transition on color properties */ .scene { transition: background ${duration}ms ease, color ${duration}ms ease; } scene.style.background = nightBg; scene.style.color = nightText; scene.textContent = 'Night Scene'; /* CSS handles the interpolation between color values automatically */

8. Motion Graphics Transitions (动态图形)

What makes YouTube/TikTok intros feel "pro". More complex, often requires canvas or WebGL.

Luma / Gradient Wipe

A black-to-white gradient is used as a mask. Dark areas reveal first, light areas last.

When to use: Organic-feeling reveals, smoke/cloud wipes, premium content.

Scene AMasked
Scene BRevealed
1000ms
/* Gradient Wipe: animate mask-image position */ .scene-b { -webkit-mask-image: linear-gradient(to right, black 0%, transparent 50%, transparent 100%); } // Animate by shifting the gradient stops: // At start: black 0%, transparent 0% (nothing visible) // At end: black 100%, transparent 150% (fully visible) // This creates a soft-edge wipe effect.

Glitch

Digital glitch effect: RGB split, slice displacement, noise. Disrupts the image briefly.

When to use: Tech content, cyberpunk aesthetic, error/disruption themes.

Scene AStable
Scene BGlitched in
400ms
// Glitch: rapid clip-path slices + color shift function glitchFrame() { const slices = 5; for (let i = 0; i < slices; i++) { const y1 = (i / slices) * 100; const y2 = ((i + 1) / slices) * 100; const offsetX = (Math.random() - 0.5) * 20; // Apply random horizontal offset per slice // Use clip-path: inset(y1% 0 (100-y2)% 0) on duplicated layers } // Add CSS filter: hue-rotate() for color shift element.style.filter = `hue-rotate(${Math.random()*90}deg)`; } // Run at 30fps for ~400ms, then hard-cut to scene B.

Particle Dissolve

Scene dissolves into particles (pixels scatter), then particles reform as the new scene.

When to use: Magical/sci-fi content, Thanos snap, premium motion graphics.

Scene AAbout to dissolve
Scene BReformed
1200ms
// Particle: requires canvas or WebGL // 1. Read pixels from scene A (or use representative color grid) // 2. Create particle for each pixel/block // 3. Animate particles: scatter with velocity + gravity // 4. After scatter peak, attract particles to scene B positions // 5. Fade in scene B, remove canvas // Simplified: fake it with random dots for (let i = 0; i < 200; i++) { const p = { x: rand(w), y: rand(h), vx: rand(-3,3), vy: rand(-3,3) }; particles.push(p); } // Animate with requestAnimationFrame

Which Transition When?

A quick decision tree. When in doubt, just cut.

  • Emotional shift? → Dissolve (cross dissolve or fade to black)
  • Same scene, time skip? → Jump cut or quick dissolve
  • New topic or section? → Wipe or push
  • Dramatic reveal? → Zoom (in or out)
  • Playful / energetic? → 3D or motion graphics
  • Branded / professional? → Dip to brand color, gradient wipe
  • Tech / cyberpunk? → Glitch
  • Not sure? → Just cut. Seriously. Hard cut is almost always correct.
