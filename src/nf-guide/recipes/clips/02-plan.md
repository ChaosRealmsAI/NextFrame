# Step 2: 挑 highlight（Agent 干）

## CLI

```bash
nf-cli source-plan <project> <episode>
# 可选：--target-count 3  (默认 3-5)
# 可选：--min-duration 30 --max-duration 90
```

## CLI 会做的事

1. 读 `<slug>/sentences.json`（271 句英文，带 start/end/text）
2. 把本 MD 的提示词 + sentences 拼成一份任务
3. 打印到 stdout（或写到 `<episode>/.plan.task.md`）
4. **等你（Agent）把结果写到** `<episode>/plan.json`
5. 校验格式（from/to 必须是合法 sentence id, to > from, duration 在窗口内）
6. 退出码 0 = 完成

## 输入

`<slug>/sentences.json`
```json
{"total_sentences": 271, "sentences": [{"id": 1, "start": 16.8, "end": 19.9, "text": "..."}]}
```

## 产出（gate: plan-written）

`<episode>/plan.json`
```json
{
  "episode": "<episode id>",
  "source": "<source slug>",
  "clips": [
    {"id": 1, "from": 1, "to": 15, "title": "开场三问", "why": "三个反差例子铺垫金句，高密度钩子"}
  ]
}
```

## 给 Agent 的提示词

你是中文短视频合集的剪辑主理人。这是一份长演讲的完整英文字幕。从中挑 3-5 段能独立传播的片段。

### 什么叫「能独立传播」

**5 个条件同时满足才算**：

1. **开头能抓人**（前 2 句内必须有一个）：疑问、反常识、具体数字、金句、反差对比。
2. **结尾有落点**：最后一句是完整句号，不是半截思考。
3. **内容独立**：单独把这段扔给没看过全片的人，他看得懂。
4. **高信息密度**：30-90 秒内至少传递 1 个清晰观点或 1 个可复述的故事。
5. **有可复述性**：看完能让人想「这句我要记下来」/「这故事我要讲给别人」。

### 5 个常见错误

| ❌ 错 | 为什么 |
|-------|-------|
| 选「引入段」（"接下来我要讲 X"） | 没内容，只是承前启后 |
| 选「结论重复段」 | 观点已在前文给过，这段只在收尾 |
| 选「充满示例但没论点」 | 听完不知道中心思想 |
| 选「讲稿卡顿/修正自己的段」 | 「uh, I mean, actually...」 节奏乱 |
| 选「依赖前面 setup 才懂的梗」 | 没上下文就 get 不到笑点 |

### 选 clip 的决策流程

1. **先扫全文**，标记所有潜在 hook 句（疑问、金句、数字）。
2. **以 hook 句为中心，向前向后延展**，直到：
   - 前面到达一个自然入口（上一句是完整收尾 / 明显话题切换）
   - 后面到达一个自然出口（下一句开始新话题 / 完整结论已给）
3. **卡时长**：若 < 30s 太短，看能不能往外扩；若 > 90s 太长，看能不能砍掉不重要的铺垫。
4. **写 title**：10 字内，**是钩子不是剧透**。
   - ❌ 「苹果为什么靠 why 成功」（剧透）
   - ✅ 「苹果凭什么？」（钩子）
5. **写 why**：20 字内告诉未来 AI 这段为什么值得剪出来。

### title 质量标准

| 原则 | 例子 |
|------|------|
| 疑问式 | 「为什么苹果能一直赢？」 |
| 数字冲击 | 「3 分钟 5 个问号」 |
| 反常识 | 「客户买的不是产品」 |
| 人名锚 | 「MLK 凭什么？」 |

**禁止**：
- 「震惊！」「必看！」
- 中文翻译腔（「关于 XX 的重要讨论」）
- 泄底（标题已给答案）

### 输出格式

**只输出这个 JSON，其他什么都不要说**：

```json
{
  "episode": "<填episode id>",
  "source": "<填source slug>",
  "clips": [
    {"id": 1, "from": <句号>, "to": <句号>, "title": "...", "why": "..."}
  ]
}
```

### 数量原则

- **宁缺毋滥**：3 个精华 > 5 个凑数
- 如果整篇质量不匀，只挑 2 个也行，但 why 要说清楚为什么整篇只有这 2 段值得
- 上限 5 个，超过说明没挑出重点

### 输入（由 CLI 附上）

你会看到 `sentences.json` 的完整内容。**只看 text 和 id，不要看 start/end**（时间不重要，code 会按 id 去算）。
