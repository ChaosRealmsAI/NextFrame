# Step 1: 定 tokens（具体值）

把 brief 里的抽象气质落成 **能直接写进组件的具体值**。

> 下一步 02-write 会把这些值落成 `theme.md`。**这一步 = 02-write 样板里每个字段的具体值**，照样板顺序定，不多不少。

## 铁律 · 极简 5 色 / 2 族 / 4 级

不要扩。
- 5 色 bg/ac/ink/ink2/rule — 多了画面会花
- 2 族 sans + (mono 或 display) — 多了风格漂移
- 4 级 H1/H2/body/cap — 多了组件会在阶梯间犹豫

## 必填清单

### A · 配色（严格 5 个）

| 变量 | 角色 | 示例 hex | 说明 |
|------|------|---------|------|
| `--bg`   | 主背景 | `#0a1628` / `#ffffff` | 占画面 80%+，不纯黑 / 不纯白皆可，但要克制 |
| `--ac`   | 主强调 | `#ff6b35` / `#2563eb` | CTA / 标题 / 高亮，画面占比 ≤ 10% |
| `--ink`  | 主文字 | `#f5f2e8` / `#1f2937` | 标题 + 正文，文字对 `--bg` contrast ≥ 7:1 |
| `--ink2` | 副文字 | `#8b92a5` / `#6b7280` | caption / metadata / 弱信息 |
| `--rule` | 分割线 | `rgba(245,242,232,0.10)` | 半透明，不抢戏 |

**来自参考物**：打开参考视频/页面截图，用取色器取 hex。**不许凭感觉写**。

**允许的扩展**（非强制，按需加 1 个）：
- `--ac2` 辅色（渐变端点 / hover）— 例：`mobile-vertical` 用了 `--ac2: #3b82f6`
- 功能色（例：代码蓝 `#58a6ff`）— 例：`blueprint-cinema` 用辅蓝显代码

禁止：一次加 3+ 个额外色；黑白混杂的灰阶一大堆。加 1 个要想清楚为什么。

### B · 字体（严格 2 个字族）

| 角色 | 字族 | 用途 |
|------|------|------|
| sans | `Inter, 'PingFang SC', system-ui, sans-serif` | 标题 + 正文 |
| mono **或** display | `'JetBrains Mono', 'SF Mono', Menlo, monospace` **或** `'PingFang SC', Inter, sans-serif` | 代码/数字 **或** 大标题/金句 |

**选 mono vs display 二选一**：
- 技术/讲解/开发者视频 → sans + mono
- 品牌/故事/vlog → sans + display

**只用系统字体 + fallback**。禁止外部 CDN — 破坏 `assets:[]` 约束。

### C · 字号阶梯（严格 4 级）

| 用途 | 16:9 (1920×1080) | 9:16 (1080×1920) 大一档 |
|------|---|---|
| H1 主标题 | 96 / 700 / 1.1 | 120 / 700 / 1.1 |
| H2 章节标题 | 56 / 600 / 1.2 | 72 / 600 / 1.2 |
| body 正文 | 28 / 400 / 1.4 | 42 / 400 / 1.5 |
| cap 注释 | 18 / 400 / 1.3 | 28 / 400 / 1.3 |

格式 `px / weight / line-height`。**竖屏要比横屏大一档**（手机观看距离近，但视口窄）。

**观看距离核校**：3m 观看最小 24px；6m 观看最小 36px。

### D · 网格（pixel-precise）

- **安全边距**：左/右/上/下 具体 px（不写"居中"）
- **主内容区**：`x ∈ [...], y ∈ [...]` 起止像素
- **chrome 区**（顶条/底条/字幕带）：各自 y 范围
- **对齐方向**：左对齐（学术/信息密度高）/ 居中（移动/品牌）/ 右对齐（数据）

算法：画布总宽 − 安全边距 × 2 = 可用宽。

### E · 图层顺序（严格 5 层）

从下到上：`bg → content → text → chrome → overlay`

对应 5 个 role 前缀：
- `bg-*` 背景（渐变、网格、纯色块）
- `content-*` 内容（图表、代码、卡片）
- `text-*` 文字（大标题、金句）
- `chrome-*` UI 外框（header、progress、字幕带）
- `overlay-*` 覆盖（聚光灯、CTA 贴纸）

声明互斥规则（例：两个 `bg-*` 不能同屏）。

| 检查 | 通过标准 |
|------|---------|

| 允许 1 个扩展色，多了 ❌ | ✓ |
| 2 字族全 fallback（禁 CDN）| ✓ |
| 4 级字号都定（H1/H2/body/cap），含 weight + 行高 | ✓ |
| 网格坐标具体（x/y 起止）| ✓ |
| 5 层 role 顺序明确 | ✓ |
