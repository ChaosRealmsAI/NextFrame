# Step 0: 主题定位 brief

定下来再动手。**模糊的定位 = 模糊的设计语言 = AI 后续乱选色。**

## 必填 8 项

1. **主题名**（kebab-case）例: `blueprint-cinema` / `mobile-vertical` / `cyberpunk-neon`
2. **ratio** 9:16 / 16:9 / 1:1 / 4:3 — 决定坐标系
3. **目标受众** — "程序员 + AI 从业者" / "30+ 白领女性" / "00 后游戏玩家"
4. **视频类型** — 访谈 / 讲解 / vlog / 产品演示 / 数据报告
5. **气质关键词** 3-5 个 — "暖棕/稳重/深夜书房" 或 "冷蓝/科技/赛博"
6. **参考物 3 个** — 具体视频 / 品牌 / 页面（不要"像 Apple"，要"像 2024 WWDC Keynote 第 12 分钟"）
7. **禁忌** — 绝对不能出现的元素（"不要花字"/"不要纯黑"/"不要 Times New Roman"）
8. **信息密度偏好** — 信息密度高（多字多图） or 留白重（一屏一要点）

## Why

| 字段 | 影响 |
|------|------|
| 目标受众 | 字号下限、配色明暗、文案严肃度 |
| 视频类型 | chrome 布局、字幕密度、镜头节奏 |
| 气质关键词 | 主色选择、字体族选择 |
| 参考物 | 所有具体值的终极真相源 |
| 禁忌 | 防止 AI 后续下意识选择 |
| 信息密度 | 网格密集度、字号阶梯跨度 |

## 怎么记录

brief 不用建新文件，**口头 / 对话里说清楚即可**，下一步 tokens 会把这些落成具体值。

如果在 autopilot 模式下，建议 append 到 `.autopilot/run-*.md` 的"项目背景"段，下一轮 AI 拿到上下文就能接。
